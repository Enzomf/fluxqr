<task type="auto">
  <name>Task 1: Deploy to Vercel via CLI with environment variables</name>
  <files></files>
  <action>
Deploy the FluxQR app to Vercel using the Vercel CLI. This is a zero-code-change task -- all work is CLI commands and external configuration.

Step 1: Check if Vercel CLI is installed. If not, install globally:
```bash
pnpm add -g vercel || npx vercel --version
```

Step 2: Link the project to Vercel (if not already linked):
```bash
vercel link
```
Follow prompts to select/create the Vercel project.

Step 3: Set ALL environment variables in Vercel for Production environment. Read values from local `.env` file and set them:
```bash
# Supabase vars
vercel env add NEXT_PUBLIC_SUPABASE_URL production
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
vercel env add SUPABASE_SERVICE_ROLE_KEY production

# Twilio vars
vercel env add TWILIO_API_KEY_SID production
vercel env add TWILIO_API_KEY_SECRET production
vercel env add TWILIO_VERIFY_SERVICE_SID production
vercel env add TWILIO_FROM_NUMBER production
```

IMPORTANT: For `NEXT_PUBLIC_SITE_URL`, the value MUST be the production URL (e.g. `https://fluxqr.vercel.app` or the custom domain). Do NOT use `http://localhost:3000`. This value is baked into the client bundle at build time and determines what URL QR codes encode.
```bash
vercel env add NEXT_PUBLIC_SITE_URL production
# Enter: https://{production-domain}
```

Step 4: Deploy to production:
```bash
vercel --prod
```

Step 5: Verify the deployment URL is accessible by checking the Vercel CLI output. Record the production URL.

NOTE: If `vercel` CLI prompts require interactive input that cannot be automated, create a checkpoint for the user. The env vars can also be set via the Vercel dashboard UI as a fallback.
