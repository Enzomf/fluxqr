1. Create `src/app/admin/page.tsx`:
   - Server Component
   - Import `createAdminClient` from `@/lib/supabase/admin`
   - Import `UserTable` from `@/components/admin/user-table`
   - Import `PageHeader` from `@/components/shared/page-header`
   - Metadata: `export const metadata = { title: 'Admin — FluxQR' }`
   - Data fetching using admin client (service role, bypasses RLS):
     a. Fetch all profiles: `const { data: profiles } = await admin.from('profiles').select('*').order('created_at', { ascending: false })`
     b. For each profile, fetch QR code aggregates. Two approaches:
        - Option A (preferred — single RPC if available): Use a Postgres function
        - Option B (simpler for MVP): Fetch all qr_codes grouped by user_id, then aggregate in JS
        - Use Option B for simplicity:
          ```typescript
          const { data: allQrCodes } = await admin.from('qr_codes').select('user_id, scan_count, is_active')
          ```
        - Aggregate in JS: for each profile, count QR codes and sum scan_counts:
          ```typescript
          const userStats = new Map<string, { qr_count: number; total_scans: number }>()
          allQrCodes?.forEach(qr => {
            if (!qr.user_id) return
            const stat = userStats.get(qr.user_id) ?? { qr_count: 0, total_scans: 0 }
            stat.qr_count++
            stat.total_scans += qr.scan_count
            userStats.set(qr.user_id, stat)
          })
          ```
        - Also include phone-only QR codes (user_id = null) stats if needed — but for the user table, these are not linked to any profile yet
     c. Merge profiles with stats into `UserRow[]` and pass to `<UserTable users={users} />`
   - Page heading: "All Users" via PageHeader component
   - Show total user count: "{N} users"

2. Create `src/app/admin/[userId]/page.tsx`:
   - Server Component
   - Import `createAdminClient` from `@/lib/supabase/admin`
   - Import `UserQrTable` from `@/components/admin/user-qr-table`
   - Import `PageHeader` from `@/components/shared/page-header`
   - Import `notFound` from `next/navigation`
   - Params: `{ params: Promise<{ userId: string }> }` (Next.js 15 async params)
   - Data fetching using admin client:
     a. Fetch profile: `const { data: profile } = await admin.from('profiles').select('*').eq('id', userId).single()`
     b. If no profile: call `notFound()`
     c. Fetch QR codes for this user: `const { data: qrCodes } = await admin.from('qr_codes').select('*').eq('user_id', userId).order('created_at', { ascending: false })`
     d. Also fetch phone-only QR codes if the profile has a phone_number:
        ```typescript
        let phoneQrCodes: QrCode[] = []
        if (profile.phone_number) {
          const { data } = await admin.from('qr_codes').select('*').eq('phone_number', profile.phone_number).is('user_id', null)
          phoneQrCodes = data ?? []
        }
        const allQrCodes = [...(qrCodes ?? []), ...phoneQrCodes]
        ```
   - Page heading: "{email}'s QR Codes" or "{phone}'s QR Codes" if no email
   - Pass to `<UserQrTable qrCodes={allQrCodes} userName={profile.email ?? profile.phone_number ?? 'Unknown'} />`
   - Generate metadata: `export async function generateMetadata({ params }) { return { title: 'User Detail — Admin — FluxQR' } }`

IMPORTANT: All admin data fetching uses the service-role admin client (createAdminClient), NOT the standard authenticated client. This is because admin needs to read across all users' data, which RLS policies restrict to own-row access.

NOTE on Next.js 15 params: In Next.js 15 App Router, dynamic route params are Promises. Use `const { userId } = await params` at the top of the function.
