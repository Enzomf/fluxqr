  <files>src/app/actions/verify-phone.ts, src/app/actions/check-otp.ts</files>
  <action>
1. Create `src/app/actions/verify-phone.ts`:
   - Add `'use server'` directive
   - Import `sendVerification` from `@/lib/twilio`
   - Import `z` from `zod`
   - Define `phoneSchema = z.string().regex(/^\+[1-9]\d{1,14}$/, 'Must include country code (e.g. +1 555 123 4567)')`
   - Export `async function sendOtp(prevState: SendOtpState, formData: FormData): Promise<SendOtpState>`
   - Extract `phone` from formData, validate with phoneSchema
   - On validation failure: return `{ error: zodError.message }`
   - On success: call `sendVerification(phone)`, return `{ success: true, phone }`
   - On Twilio error: catch and return `{ error: 'Failed to send verification code. Please try again.' }`
   - Export type `SendOtpState = { error?: string; success?: boolean; phone?: string }`

2. Create `src/app/actions/check-otp.ts`:
   - Add `'use server'` directive
   - Import `checkVerification` from `@/lib/twilio`
   - Import `cookies` from `next/headers`
   - Import `z` from `zod`
   - Define `otpSchema = z.string().length(6).regex(/^\d{6}$/, 'Must be 6 digits')`
   - Export `async function checkOtp(phone: string, code: string): Promise<CheckOtpResult>`
   - Validate code with otpSchema; on failure return `{ error: 'Invalid code format' }`
   - Call `checkVerification(phone, code)`
   - If `verification.status !== 'approved'`: return `{ error: 'Invalid or expired code. Please try again.' }`
   - If approved: set httpOnly cookie:
     ```typescript
     const cookieStore = await cookies()
     cookieStore.set('verified_phone', phone, {
       httpOnly: true,
       secure: process.env.NODE_ENV === 'production',
       sameSite: 'lax',
       maxAge: 60 * 60 * 24 * 30, // 30 days
       path: '/',
     })
     ```
   - Return `{ success: true }`
   - On Twilio error: catch and return `{ error: 'Verification failed. Please try again.' }`
   - Export type `CheckOtpResult = { error?: string; success?: boolean }`

NOTE: checkOtp is NOT a form action (not using prevState/formData pattern) because it is called programmatically from the OTP component on auto-submit when 6 digits are entered. Use a plain async function signature with direct params.
