<task type="auto">
  <name>Task 1: Create middleware for dashboard and admin route protection</name>
  <files>src/middleware.ts</files>
  <action>
Create `src/middleware.ts` (note: Next.js middleware must be at root of src/ or project root — this project uses src/):

1. Import `createServerClient` from `@supabase/ssr`, `NextResponse` and `NextRequest` from `next/server`

2. Export `async function middleware(request: NextRequest)`:
   a. Create a mutable response: `let response = NextResponse.next({ request: { headers: request.headers } })`
   b. Create Supabase server client with cookie bridge (read from request, write to response):
      ```typescript
      const supabase = createServerClient(
        process.env.NEXT_PUBLIC_SUPABASE_URL!,
        process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
        {
          cookies: {
            getAll() { return request.cookies.getAll() },
            setAll(cookiesToSet) {
              cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value))
              response = NextResponse.next({ request })
              cookiesToSet.forEach(({ name, value, options }) =>
                response.cookies.set(name, value, options)
              )
            },
          },
        }
      )
      ```
   c. Refresh session: `const { data: { user } } = await supabase.auth.getUser()`

   d. Dashboard protection (`/dashboard/*`):
      - If no user: redirect to `/login`

   e. Admin protection (`/admin/*`):
      - If no user: redirect to `/login`
      - If user exists: query profiles for role:
        ```typescript
        const { data: profile } = await supabase
          .from('profiles')
          .select('role')
          .eq('id', user.id)
          .single()
        ```
      - If `profile?.role !== 'admin'`: redirect to `/dashboard`

   f. Return response

3. Export matcher config:
   ```typescript
   export const config = {
     matcher: ['/dashboard/:path*', '/admin/:path*'],
   }
   ```

IMPORTANT: The Supabase middleware client pattern uses request.cookies for getAll and response.cookies for setAll. This is the standard @supabase/ssr middleware pattern that correctly handles cookie refresh during session checks. Do NOT import from src/lib/supabase/server.ts — middleware needs its own client creation.

NOTE on defense-in-depth: The existing dashboard layout (src/app/dashboard/layout.tsx) already has a getUser() + redirect guard. The new middleware adds a second layer. Both guards are intentional — middleware runs earlier (before layout), layout provides a fallback. This defense-in-depth pattern is standard for Supabase + Next.js apps.

1. Extend `src/app/auth/callback/route.ts`:
   - Keep ALL existing code (inline Supabase client creation, exchangeCodeForSession, cookie apply, redirect to /dashboard)
   - AFTER `exchangeCodeForSession` succeeds, add account linking logic:
     a. Get the user from the session: `const { data: { user } } = await supabase.auth.getUser()`
     b. Read verified_phone cookie: `const verifiedPhone = cookieStore.get('verified_phone')?.value`
     c. If `verifiedPhone && user`:
        - Import and call `createAdminClient` from `@/lib/supabase/admin`
        - Link phone QR codes to user:
          ```typescript
          await admin.from('qr_codes')
            .update({ user_id: user.id })
            .eq('phone_number', verifiedPhone)
            .is('user_id', null)
          ```
        - Update profile with phone number:
          ```typescript
          await admin.from('profiles')
            .update({ phone_number: verifiedPhone })
            .eq('id', user.id)
          ```
        - Delete the verified_phone cookie (user is now fully authenticated):
          ```typescript
          cookieStore.delete('verified_phone')
          ```
          Also set it on the response: `response.cookies.delete('verified_phone')`
     d. These operations are fire-and-forget for redirect speed — wrap in try/catch, log errors but don't block redirect
   - The redirect destination stays as /dashboard

2. Update `src/components/dashboard/sidebar.tsx`:
