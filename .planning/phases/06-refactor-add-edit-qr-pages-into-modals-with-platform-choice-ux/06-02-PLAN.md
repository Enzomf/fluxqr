Create `src/components/qr-management/qr-form-dialog.tsx` as a `'use client'` component.

**Props interface:**
```typescript
interface QrFormDialogProps {
  open: boolean
  onOpenChange: (open: boolean) => void
  qr?: QrCodeWithImage | null      // if provided = edit mode
  verifiedPhone: string | null
  onEditSuccess?: (id: string) => void  // for pulse animation
}
```

**Internal state:**
- `step: 'grid' | 'form'` — initialized to `'form'` if `qr` is provided (edit mode), `'grid'` otherwise (create mode)
- `qrType: 'default' | 'custom'` — set by grid selection; for edit mode, derive from `qr.default_message`: if `qr.default_message` is truthy, set `'custom'`; otherwise `'default'`

**Reset on close:** In `handleOpenChange(next: boolean)`, when `next` is false, reset `step` and `qrType` to their initial values based on mode. This ensures form state is clean on re-open.

**Key insight for form state reset:** Pass `key={open ? (qr?.id ?? 'create') : 'closed'}` to the `QrForm` component so `useActionState` resets between opens. This prevents stale validation errors from lingering.

**Bound action for edit:** When in edit mode, use `updateQrCode.bind(null, qr!.id)` as the action. For create, use `createQrCode` directly.

**onSuccess handler:** Create a handler that:
- Closes the dialog via `onOpenChange(false)`
- If edit mode and `onEditSuccess` callback exists, call `onEditSuccess(qr!.id)` for pulse animation

Note: toast and router.refresh are handled by QrForm's own onSuccess/useEffect (from Plan 01), so QrFormDialog only needs to close itself and signal the pulse ID.

**Layout — use shadcn Dialog with override classes on DialogContent:**
- Override `sm:max-w-sm` to `sm:max-w-lg` on DialogContent className
- Use `flex flex-col max-h-[85vh] p-0 gap-0` on DialogContent to enable sticky header/footer
- Set `showCloseButton={false}` on DialogContent (we'll include the X in the custom header)
