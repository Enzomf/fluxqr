**src/app/dashboard/[id]/edit/actions.ts** — ADD `updateQrCode` to the existing file (which already has `deleteQrCode` from Plan 03):

Read the existing file first, then add:

Define `UpdateQrSchema` using Zod:
- `label`: `z.string().min(1, 'Label is required').max(100, 'Label too long')`
- `slug`: `z.string().min(1, 'Slug is required').regex(/^[a-z0-9-]+$/, 'Only lowercase letters, numbers, and hyphens').min(2).max(50)`
- `contact_target`: `z.string().min(1, 'Contact target is required')`
- `default_message`: `z.string().optional().default('')`
- NOTE: NO `platform` field — platform is read-only after creation (project rule). Exclude it from validation and update entirely.

Export `updateQrCode(id: string, prevState: FormState, formData: FormData): Promise<FormState>`:
1. Parse form data with `UpdateQrSchema.safeParse(Object.fromEntries(formData))`.
2. If invalid, return `{ errors: validated.error.flatten().fieldErrors }`.
3. Call `createClient()`, then `supabase.auth.getUser()`. If no user, `redirect('/login')`.
4. Update: `supabase.from('qr_codes').update({ label, slug, contact_target, default_message }).eq('id', id).eq('user_id', user.id)`.
5. If error code `'23505'`, return `{ errors: { slug: ['This slug is already taken'] } }`.
6. If any other error, return `{ message: 'Failed to update QR code. Please try again.' }`.
7. Call `revalidatePath('/dashboard')`.
8. Call `redirect('/dashboard?success=edit&id=${id}')` — outside try/catch. Include BOTH `success=edit` for the toast AND `id=${id}` so the list knows which row to pulse.

IMPORTANT: The `updateQrCode` action has THREE parameters (id, prevState, formData). Since `useActionState` only passes (prevState, formData), the edit page must use `.bind(null, id)` to partially apply the id. See page.tsx below.
