**src/lib/qr-generator.ts** — QR image generation (server-side) and PNG download (client-side):
- `generateQrDataUrl(slug: string): Promise<string>` — calls `QRCode.toDataURL()` with the URL `${process.env.NEXT_PUBLIC_SITE_URL}/q/${slug}`, brand colors (dark: `#0F172A`, light: `#FFFFFF`), errorCorrectionLevel `'H'`, width 400, margin 2. Returns the data URL string.
- `downloadQrPng(dataUrl: string, slug: string): void` — client-only function that creates an anchor element, sets `href` to dataUrl, `download` to `${slug}-fluxqr.png`, and triggers click. This function must NOT import `qrcode` — it only manipulates the DOM.

IMPORTANT: Do NOT put `'use client'` or `'use server'` on this file. `generateQrDataUrl` is called from Server Components. `downloadQrPng` is called from Client Components that import it directly (tree-shaking handles it because `downloadQrPng` only uses `document`).

**src/app/api/slug-check/route.ts** — GET handler for slug availability:
- Extract `slug` from `request.nextUrl.searchParams`. If missing, return `Response.json({ available: false })`.
- Use `createClient()` from `@/lib/supabase/server` (anon key — RLS allows public SELECT of active rows).
- Query `qr_codes` for `.select('id').eq('slug', slug).eq('is_active', true).single()`. If data exists, slug is taken; if null, available.
- Return `Response.json({ available: boolean })`.

**src/hooks/use-slug-check.ts** — debounced slug validation hook:
- `'use client'` directive at top.
- Type: `SlugStatus = 'idle' | 'checking' | 'available' | 'taken' | 'invalid'`
- Accept `slug: string` and optional `currentSlug?: string` (for edit mode — skip check if slug matches current).
- If slug is empty, return `'idle'`. If it matches `currentSlug`, return `'available'` immediately.
- Validate format with `/^[a-z0-9-]+$/` — if invalid, return `'invalid'`.
- Set `'checking'`, debounce 300ms via `useRef<ReturnType<typeof setTimeout>>`, then fetch `/api/slug-check?slug=${slug}`.
- Cleanup timeout on unmount and slug change.
- Return `status` as `SlugStatus`.

**src/hooks/use-qr-image.ts** — client-side QR data URL generation for download:
- `'use client'` directive.
- Accept `slug: string` and `dataUrl: string` (the server-generated data URL passed as prop).
- This hook simply returns `{ dataUrl, download: () => downloadQrPng(dataUrl, slug) }`.
- Import `downloadQrPng` from `@/lib/qr-generator`.
- This hook exists so Client Components have a clean API for download without importing the generator directly.

**src/components/shared/page-header.tsx** — reusable page header:
- Server Component (no `'use client'`).
- Props: `title: string`, optional `description?: string`, optional `action?: { label: string; href: string }`.
- Render: `<div>` with flex justify-between. `<h1>` with `text-2xl font-bold`. Optional `<p>` with `text-muted-foreground`. If `action`, render a Link styled as brand button (bg-brand-500 hover:bg-brand-600 text-white rounded-md px-4 py-2 flex items-center gap-2) with a Plus icon from lucide-react.
- Use `cn()` for class merging.

**src/components/shared/platform-badge.tsx** — colored platform badge:

- Props: `platform: Platform` (import from `@/types`).
- Use the shadcn Badge component. Map platform to colors: whatsapp -> green (`bg-emerald-500/10 text-emerald-400 border-emerald-500/20`), sms -> blue (`bg-blue-500/10 text-blue-400 border-blue-500/20`), telegram -> sky (`bg-sky-500/10 text-sky-400 border-sky-500/20`).
