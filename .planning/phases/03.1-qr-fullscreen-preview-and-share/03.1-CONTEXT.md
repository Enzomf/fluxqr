# Phase 03.1: QR Fullscreen Preview & Share - Context

**Gathered:** 2026-03-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Clicking a QR code thumbnail in the dashboard list opens a fullscreen preview dialog with the QR displayed at large size, plus share actions (Web Share API and copy link). Does not include editing, deleting, analytics detail, or scan history from the preview.

</domain>

<decisions>
## Implementation Decisions

### Preview trigger
- Click the 40px QR thumbnail in QrListRow to open the preview — not the row, not a new button
- Thumbnail becomes a clickable target with cursor-pointer and hover state
- Row action buttons (edit, download, delete) remain independent — no interaction conflicts

### Container
- shadcn Dialog modal — install dialog.tsx component
- Same dialog on all screen sizes (responsive, no Sheet on mobile)
- Dark blur backdrop (backdrop-blur + dark overlay) for immersive feel

### QR display
- QR image rendered at ~280px — large enough to scan from screen
- QR floats directly on the dark dialog surface (no white card frame — the QR already has a white background from generation)
- Centered vertical stack below QR: label, platform badge + scan count on a row
- No slug URL shown — label, PlatformBadge component, and scan count only

### Share actions
- Two actions below metadata: Share (Web Share API) and Copy link
- Share button only visible when `navigator.share` is available — hidden on unsupported browsers
- Web Share shares link only (URL + title) — no image file attachment
- Copy link copies full QR URL (e.g. `https://fluxqr.com/q/biz`) to clipboard
- Copy feedback: button text changes to "Copied!" with check icon for ~2s, then reverts — no toast
- Create a `use-copy-to-clipboard` hook for the copy logic

### Animation
- Open: QR grows from the thumbnail's screen position to 280px centered in dialog — CSS transform-origin + scale, ~300ms ease-out, backdrop fades in simultaneously
- Close: reverse animation — QR shrinks back toward the thumbnail position
- All dismiss methods (X button, backdrop click, Escape) trigger the reverse animation
- Requires tracking the thumbnail element ref to compute start/end positions
- No Framer Motion — Tailwind keyframes + inline CSS transforms only

### Dismiss
- X button in top-right corner of dialog
- Click backdrop to close
- Escape key (built-in with shadcn Dialog)
- All three trigger the reverse shrink animation

### Claude's Discretion
- Exact easing curves and animation timing refinements
- Dialog padding and spacing values
- Action button styling (icon-only vs icon+text, ghost vs outline variant)
- How to compute and cache thumbnail position for animation origin
- Responsive sizing adjustments for the QR on very small screens

</decisions>
