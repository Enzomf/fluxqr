  <action>
Create `src/components/dashboard/qr-preview-dialog.tsx` as a `'use client'` component.

**Props interface:**
```typescript
interface QrPreviewDialogProps {
  open: boolean
  onOpenChange: (open: boolean) => void
  qr: QrCodeWithImage
  thumbnailRect: DOMRect | null  // captured at click time from thumbnail ref
}
```

Import `QrCodeWithImage` from `./qr-list-row`.

**Component structure:**

1. **Transform-origin computation:** When `open` is true and `thumbnailRect` is provided, compute the element-relative transform-origin for the QR image animation. The dialog popup is `position: fixed; top: 50%; left: 50%; transform: translate(-50%, -50%)`. For a centered fixed element, the QR image (280x280) center in viewport is at `(vw/2, vh/2)`. The thumbnail center is at `(thumbnailRect.left + thumbnailRect.width/2, thumbnailRect.top + thumbnailRect.height/2)`. Element-relative origin for the QR image = `(thumbCenterX - (vw/2 - 140), thumbCenterY - (vh/2 - 140))` where 140 = 280/2. Store this as a `transformOrigin` string using `useState`, computed in an effect that runs when `open` changes to `true`. Capture the rect in the onClick handler in QrListRow (not in a useEffect) — the thumbnailRect prop already contains the fresh DOMRect.

2. **Web Share guard:** `const canShare = typeof navigator !== 'undefined' && 'share' in navigator`

3. **Share handler:** Async function that calls `navigator.share({ title: qr.label + ' — FluxQR', url: siteUrl + '/q/' + qr.slug })` wrapped in try/catch. Ignore `AbortError` (user cancelled share sheet), log other errors.

4. **Site URL:** `const siteUrl = process.env.NEXT_PUBLIC_SITE_URL ?? (typeof window !== 'undefined' ? window.location.origin : '')`

5. **Copy hook:** `const { copied, copy } = useCopyToClipboard()`

6. **JSX structure:**
```
<Dialog open={open} onOpenChange={onOpenChange}>
  <DialogPortal>
    <DialogOverlay className="..." />  -- dark blur backdrop: bg-black/60 backdrop-blur-sm
    <DialogContent className="...">    -- override default classes for this use case
      <DialogTitle className="sr-only">{qr.label} QR Preview</DialogTitle>

      <!-- Close button: top-right X icon -->
      <DialogClose className="...">
        <X size={20} />
      </DialogClose>

      <!-- QR Image with pop animation -->
      <div className="flex flex-col items-center gap-4">
        <img
          src={qr.dataUrl}
          alt={qr.label}
          width={280}
          height={280}
          className="rounded-lg"
          style={{
            transformOrigin: transformOrigin ?? 'center',
            animation: open
              ? 'qr-pop-open 300ms ease-out forwards'
              : 'qr-pop-close 200ms ease-in forwards'
          }}
        />

        <!-- Metadata: label, badge + scan count -->
        <div className="text-center space-y-2">
          <p className="text-lg font-semibold">{qr.label}</p>
          <div className="flex items-center justify-center gap-2">
            <PlatformBadge platform={qr.platform} />
            <span className="text-sm text-muted-foreground">
              {formatScanCount(qr.scan_count)} scans
            </span>
          </div>
        </div>

        <!-- Share actions -->
        <div className="flex items-center gap-3 mt-2">
          {canShare && (
            <button onClick={handleShare} className="...ghost button with Share2 icon...">
              <Share2 size={16} />
              Share
            </button>
          )}
          <button onClick={() => copy(siteUrl + '/q/' + qr.slug)} className="...ghost button...">
            {copied ? <Check size={16} /> : <Link2 size={16} />}
            {copied ? 'Copied!' : 'Copy link'}
          </button>
        </div>
      </div>
    </DialogContent>
  </DialogPortal>
</Dialog>
```

7. **DialogContent className overrides:** Override the default shadcn DialogContent classes for the preview use case:
   - Keep `fixed top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2 z-50`
   - Set `max-w-sm w-auto p-6 rounded-xl`
   - Use `bg-background ring-1 ring-foreground/10`
   - Override duration to `duration-300` so @base-ui keeps the popup mounted long enough for the close animation
   - Keep `data-open:animate-in data-open:fade-in-0` and `data-closed:animate-out data-closed:fade-out-0` for the container fade
   - Remove the default `zoom-in-95` / `zoom-out-95` — the QR image handles its own scale animation via inline style

8. **DialogOverlay className:** `bg-black/60 backdrop-blur-sm` for the immersive dark blur effect. Keep `data-open:animate-in data-open:fade-in-0 data-closed:animate-out data-closed:fade-out-0` and add `duration-300`.

9. **Action button styling (Claude's discretion):** Use ghost-style buttons with `inline-flex items-center gap-2 px-4 py-2 rounded-md text-sm font-medium text-muted-foreground hover:text-foreground hover:bg-accent transition-colors`. The "Copied!" state button gets `text-success` (emerald).

10. **Close button styling:** Absolute positioned `top-3 right-3`, ghost style, `text-muted-foreground hover:text-foreground`.

11. **Icons:** Import `X`, `Share2`, `Check`, `Link2` from `lucide-react` (already installed).

12. **Responsive:** On small screens (below ~320px width), the 280px QR might be tight. Use `w-[280px] max-w-[calc(100vw-4rem)]` and matching height with `aspect-square` so it scales down gracefully.


   ```typescript

       setThumbnailRect(thumbnailRef.current.getBoundingClientRect())
     }
     setPreviewOpen(true)
   }
   ```

   IMPORTANT: Capture the rect at click time (before dialog opens) — this gives the fresh viewport position of the thumbnail even if the user has scrolled.

   - Add `role="button"` and `tabIndex={0}` for keyboard accessibility
   - Add `onKeyDown` handler: open on Enter or Space key

   Updated img:
   ```tsx
   <img
     ref={thumbnailRef}
     src={qr.dataUrl}
     alt={qr.label}
     width={40}
     height={40}
     className="rounded shrink-0 cursor-pointer hover:opacity-80 transition-opacity"

     tabIndex={0}
     onClick={handleThumbnailClick}
   ```
