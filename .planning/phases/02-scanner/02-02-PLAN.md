    **Create `src/app/q/[slug]/page.tsx`** (async Server Component, no "use client"):

    Imports:
    - `after` from `next/server`
    - `notFound` from `next/navigation`
    - `createClient` from `@/lib/supabase/server`
    - `createServerClient` from `@supabase/ssr` (for the after() callback and inactive check)
    - `ScannerLanding` from `./scanner-landing`

    Export metadata:
    ```typescript
    export const metadata = { title: 'FluxQR' }
    ```

    Default export: `async function ScannerPage({ params }: { params: Promise<{ slug: string }> })`

    Implementation flow:
    1. `const { slug } = await params` -- MUST await params (Next.js 15+ requirement)
    2. `const supabase = await createClient()`
    3. Query active QR record using anon client:
       ```typescript
       const { data: qr } = await supabase
         .from('qr_codes')
         .select('id, user_id, slug, label, platform, contact_target, default_message, is_active, scan_count, created_at, updated_at')
         .eq('slug', slug)
         .eq('is_active', true)
         .single()
       ```
    4. If `qr` is null -- distinguish missing vs inactive. The anon RLS policy `public_select_active` only allows SELECT where `is_active = true`, so anon cannot read inactive rows. Use a service-role client (server-side only, safe in Server Component) to check if the slug exists at all:
       ```typescript
       if (!qr) {
         const adminClient = createServerClient(
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.SUPABASE_SERVICE_ROLE_KEY!,
           {
             cookies: {
               getAll() { return [] },
               setAll() {},
             },
           }
         )
         const { data: existing } = await adminClient
           .from('qr_codes')
           .select('id')
           .eq('slug', slug)
           .single()

         if (existing) {
           return (
             <main className="min-h-screen flex items-center justify-center bg-surface">
               <div className="bg-surface-raised rounded-lg p-8 w-full max-w-sm text-center">
                 <p className="text-foreground">This link has been deactivated.</p>
               </div>
             </main>
           )
         }

         notFound()
       }
       ```
       Per CONTEXT.md locked decisions: missing slug shows "This link does not exist" (via not-found.tsx), inactive slug shows "This link has been deactivated" (inline render). Using service role key server-side is safe -- it never reaches the client.

    5. Fire-and-forget scan count increment using `after()`:
       ```typescript
       after(async () => {
         const supabase = createServerClient(
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
           {
             cookies: {
               getAll() { return [] },
               setAll() {},
             },
           }
         )
         await supabase.rpc('increment_scan_count', { qr_slug: slug })
       })
       ```
       CRITICAL: Do NOT call `createClient()` from `@/lib/supabase/server` inside `after()` -- it calls `cookies()` which throws inside `after()` callbacks. Create a minimal Supabase client from `@supabase/ssr` with empty cookie handlers. The RPC is SECURITY DEFINER so no auth is needed.

    6. Return `<ScannerLanding qr={qr} />`
