1. **Create src/app/dashboard/actions.ts** — signOut server action:
   ```typescript
   'use server'
   import { createClient } from '@/lib/supabase/server'
   import { redirect } from 'next/navigation'

   export async function signOut() {
     const supabase = await createClient()
     await supabase.auth.signOut()
     redirect('/login')
   }
   ```

2. **Create src/components/dashboard/sidebar-link.tsx** — Client Component:
   - Mark with `'use client'`
   - Import `Link` from `next/link`, `usePathname` from `next/navigation`, `cn` from `@/lib/utils`
   - Accept props: `{ href: string; label: string; icon?: React.ReactNode }`
   - Determine active state: `pathname === href || pathname.startsWith(\`${href}/\`)`
   - Active styles: `bg-brand-500/10 text-brand-400 border-l-2 border-brand-500`
   - Inactive styles: `text-slate-400 hover:text-slate-200 hover:bg-surface-raised`
   - Use `cn()` for conditional class merging
   - Follow RESEARCH.md active sidebar link pattern exactly

3. **Create src/components/dashboard/sidebar.tsx** — Client Component:
   - Mark with `'use client'`
   - Accept prop: `{ user: { email?: string; user_metadata?: { avatar_url?: string; full_name?: string } } }`
   - Define nav items array: `[{ href: '/dashboard', label: 'My QR Codes' }]` — ONLY one item per user decision

   **Shared SidebarNav internal component** (renders the nav content, used in both desktop and mobile):
   - Top section: FluxQR wordmark (`text-brand-500 font-bold text-xl`)
   - Middle section (flex-1): Map over navItems, render `<SidebarLink>` for each
   - Bottom section (separated by `<Separator>`):
     - Avatar: Use shadcn `<Avatar>` with `<AvatarImage>` using `user.user_metadata?.avatar_url`. `<AvatarFallback>` shows first letter of email or name
     - Email: truncated with `truncate` class, `text-sm text-muted-foreground`, max-width constrained
     - Sign-out button: `<form action={signOut}>` with a `<Button variant="ghost" size="sm">` labeled "Sign out"
     - Import `signOut` from `@/app/dashboard/actions`

   **Desktop sidebar:**
   - `<aside className="hidden md:flex md:w-56 md:flex-col md:fixed md:inset-y-0 bg-surface-raised border-r border-border">`
   - Renders `<SidebarNav />` inside

   **Mobile sidebar:**
   - Fixed hamburger button: `<button className="fixed top-4 left-4 z-50 md:hidden">` with `<Menu />` icon from lucide-react
   - shadcn `<Sheet>` wrapping the hamburger as `<SheetTrigger>`
   - `<SheetContent side="left" className="w-56 bg-surface-raised p-0">` renders `<SidebarNav />`
   - Use `useState` for controlled open/close of Sheet. Close Sheet when a nav link is clicked (pass `onNavigate` callback to SidebarLink or use Sheet's `onOpenChange`).

   IMPORTANT: The sidebar renders the same SidebarNav content in both desktop and mobile — DRY principle.
