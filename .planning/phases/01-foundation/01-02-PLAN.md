2. **Create supabase/migrations/0001_create_qr_codes.sql** with the complete migration from RESEARCH.md Pattern 7. The file must include ALL of the following in this order:

   a. **CREATE TABLE public.qr_codes** with columns:
      - `id` uuid DEFAULT gen_random_uuid() PRIMARY KEY
      - `user_id` uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
      - `slug` text NOT NULL UNIQUE
      - `label` text NOT NULL
      - `platform` text NOT NULL CHECK (platform IN ('whatsapp', 'sms', 'telegram'))
      - `contact_target` text NOT NULL
      - `default_message` text (nullable)
      - `is_active` boolean NOT NULL DEFAULT true
      - `scan_count` integer NOT NULL DEFAULT 0
      - `created_at` timestamptz NOT NULL DEFAULT now()
      - `updated_at` timestamptz NOT NULL DEFAULT now()

   b. **Partial index** for proxy lookup performance:
      ```sql
      CREATE INDEX idx_qr_codes_slug_active ON public.qr_codes (slug) WHERE is_active = true;
      ```

   c. **update_updated_at() trigger function** and trigger on qr_codes:
      ```sql
      CREATE OR REPLACE FUNCTION public.update_updated_at()
      RETURNS TRIGGER LANGUAGE plpgsql AS $$
      BEGIN
        NEW.updated_at = now();
        RETURN NEW;
      END;
      $$;

      CREATE TRIGGER qr_codes_updated_at
        BEFORE UPDATE ON public.qr_codes
        FOR EACH ROW EXECUTE FUNCTION public.update_updated_at();
      ```

   d. **Enable RLS:**
      ```sql
      ALTER TABLE public.qr_codes ENABLE ROW LEVEL SECURITY;
      ```

   e. **Owner CRUD policies** (4 separate policies for SELECT, INSERT, UPDATE, DELETE):
      - All use `auth.uid() = user_id` in USING/WITH CHECK clauses
      - UPDATE policy needs both USING and WITH CHECK

   f. **Public SELECT policy** for scanner proxy:
      ```sql
      CREATE POLICY "public_select_active" ON public.qr_codes
        FOR SELECT USING (is_active = true);
      ```
      Note: This intentionally allows unauthenticated reads of active codes. The scanner proxy needs this. Postgres RLS uses OR logic — either the owner policy or the public policy passes.

   g. **Atomic scan count RPC:**
      ```sql
      CREATE OR REPLACE FUNCTION public.increment_scan_count(qr_slug text)
      RETURNS void LANGUAGE plpgsql SECURITY DEFINER AS $$
      BEGIN
        UPDATE public.qr_codes
        SET scan_count = scan_count + 1
        WHERE slug = qr_slug AND is_active = true;
      END;
      $$;
      ```

Add clear SQL comments separating each section (table, index, trigger, RLS, RPC).
