  <action>
1. Install all test dependencies using pnpm (the project uses pnpm, not npm):
   ```
   pnpm add -D vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/dom @testing-library/jest-dom @testing-library/user-event vite-tsconfig-paths
   ```

2. Create `vitest.config.mts` at project root following the official Next.js 16 Vitest guide pattern:
   - `plugins: [tsconfigPaths(), react()]` — tsconfigPaths FIRST for @/* alias resolution
   - `test.environment: 'jsdom'`
   - `test.globals: true` — auto-import describe/it/expect/vi
   - `test.setupFiles: ['./src/test/setup.ts']`
   - `test.coverage.provider: 'v8'`
   - `test.coverage.include: ['src/**/*.{ts,tsx}']`
   - `test.coverage.exclude: ['src/components/ui/**', 'src/app/**', 'src/types/**', 'src/test/**', '**/*.test.{ts,tsx}', '**/*.spec.{ts,tsx}']`

3. Add scripts to `package.json`:
   - `"test": "vitest"`
   - `"test:run": "vitest run"`
   - `"test:coverage": "vitest run --coverage"`

4. Update `tsconfig.json`: add `"vitest/globals"` to the `compilerOptions.types` array (create the array if it does not exist). This provides global type definitions for describe, it, expect, vi without explicit imports.

5. Create `src/test/setup.ts` with global mocks:
   - `import '@testing-library/jest-dom/vitest'` (Vitest-compatible matchers)
   - `afterEach(() => { cleanup(); vi.clearAllMocks() })`
   - Mock `next/navigation` globally: useRouter (returns object with push/replace/refresh/back/prefetch as vi.fn()), usePathname (returns '/'), useSearchParams (returns new URLSearchParams())
   - Mock `next/image` globally: return plain `<img>` element (avoids Image Optimization API errors in jsdom)
   - Mock `navigator.clipboard` on globalThis: writeText as vi.fn().mockResolvedValue(undefined), readText as vi.fn().mockResolvedValue('')
   - Mock `next/link` globally: return plain `<a>` element that passes href and children through (avoids Next.js router context requirement)

6. Create `src/test/utils.tsx` as a minimal re-export of RTL render. If providers are needed later, this is the place to add them:
   ```typescript
   export { render, screen, waitFor } from '@testing-library/react'
   export { default as userEvent } from '@testing-library/user-event'
   ```

7. Verify setup works by running `pnpm test:run` (should pass with 0 tests found, no errors).
